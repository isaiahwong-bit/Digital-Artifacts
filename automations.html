01 Why a framework

Prompt-only AI is a black box.

Most teams pay for ChatGPT and hope for the best. One prompt, no memory, no audit trail. It drifts, hallucinates, and six months in nobody trusts its output. The fix isn't a better prompt. It's a better architecture.

Four layers. Each one is inspectable. Each one is fixable. Each one stays the same when you swap models, vendors, or interfaces.

Layer 01 / the brain

Context.

01/

Your business, written down. Structured markdown in a Git repo you own. Human-editable. AI-readable. Version-controlled forever.

It's the stable ground the rest of the stack stands on. Positioning, clients, pipeline, SOPs, tone of voice, team roster, pricing logic, competitor battlecards. Everything your team already "just knows", made explicit and queryable.

Humans write, AI reads. The brain is only edited through a review step. Never mutated by the model at runtime.

Correct a fact once. Every brief, draft, and summary from that day forward is right.

Portable forever. Swap Claude for a different model tomorrow, the brain doesn't change.

brain / company-brief.md 
---
name: Company brief
updated: 2026-04-21
---

# Who we are
We build AI operating systems for small business.
One brain, many surfaces. Layers over prompts.

# Who we serve
## Primary
- Professional services firms (legal, accounting, consulting)
- Trades & field services (plumbing, electrical, building)
- B2B sales teams (agencies, SaaS)

# Tone of voice
Professional, warm, never salesy. Peer-to-peer.
No em dashes. Short sentences. Direct.

# Non-negotiables
- Human review before any AI-written commit
- Every output cites the brain file it came from
- Brief beats fluff. Always.

Sample. Real deployments run 20-60 files across `clients/`, `pipeline/`, `skills/`, `tone-of-voice/`, and more.

Layer 02 / the how

Skills.

02/
skills / proposal-writer.md 
---
name: proposal-writer
reads:
  - clients/{client}.md
  - pricing.md
  - tone-of-voice.md
output: docx
---

# When to use
A client has asked for a proposal. You want a
first draft ready for review in 60 seconds.

# The prompt
Draft a proposal for {{ client }} covering
{{ scope }}. Use the firm's standard format.
Pull pricing from pricing.md. Match tone from
tone-of-voice.md. End with a clear next step.

Sample. A typical deploy has 5-15 skills, each 30-100 lines, each knowing exactly which parts of the brain to open.

Reusable recipes for the jobs your team does every week. Drafting a proposal. Prepping for a meeting. Triaging an inbox. Writing a follow-up. Each skill is a short, named file.

Think of them as saved searches crossed with templates crossed with standard operating procedures. Each one declares what it reads from Layer 1 and what it produces. That means outputs are consistent, reviewable, and portable across whoever is using them.

Named and versioned. `proposal-writer-v2` is a Git commit away. Never lose a prompt that worked.

Composable. A `follow-up` skill can call a `client-update` skill, which calls `tone-of-voice`. Layers all the way down.

Your voice, not the model's. Skills pin the output format so drafts sound like you, not ChatGPT.

Layer 03 / the triggers

Automation.

03/

Scheduled and event-driven workflows that tie Context and Skills to the rest of your stack. n8n runs the wiring.

Cron jobs that fire at 8:30am. Webhooks that catch a new lead the instant it lands. API calls that pull a Fireflies transcript and drop a draft meeting note in your inbox. Conditional branches, retries, error alerts. All visual. All inspectable. All yours to edit after we hand it over.

cron: daily 08:30 webhook: new-lead trigger: transcript-ready retry: 3x exponential

A real n8n workflow analysing 100 audits, scoring results, delivering a summary. No clicks.

Daily brief at 8:30am

Cron pulls today's matters, deadlines, and inbound. Claude drafts the brief. Slack delivers it. One fewer 6am scramble.

New lead triage

Webhook fires on form submit. Classify by ICP, enrich with competitor context, write to HubSpot, ping Slack. 3 seconds end-to-end.

Approval flow

Draft reply to a client. Post to Slack with approve/reject buttons. On approve: send via Outlook, log to CRM, confirm in thread.

Layer 04 / the face

Agent.

04/

How your team talks to the system and how the system talks back. Slack, SMS, voice, dashboards, email, whatever fits where the team already works.

One brain, many surfaces. The operator in the field gets SMS. The partner gets a daily email. The rep uses Slack. The CEO asks by voice while driving. Same underlying system, different face depending on who's asking and what they need.

Slack

DM a Slack bot "what's the status on Acme?" Get a 3-second synthesis with links to every source file.

SMS

Crew gets the day's schedule at 6am. A missed invoice fires a follow-up sequence. No app to download.

Voice (ElevenLabs + Twilio)

Call a number, ask a question, get a synthesised voice answer. Missed calls get triaged and logged.

Dashboard

Custom web interface for the office. Open pipeline, job queue, team utilisation, rendered live from the brain.

06 How we build it

Five steps. No mystery.

Most engagements run 4-8 weeks to initial deployment. We stay on as a retainer to grow the brain, add skills, and wire up new surfaces as your team uses them.

Workflow audit

We sit with your team for a day or two. Map what everyone actually does each week. Find the five repeatable jobs worth automating first.

Brain seed (Layer 1)

We set up the Git repo, write the first 15-25 context files with you, get the structure right. You own the repo from day one.

Skills authoring (Layer 2)

We build 3-5 skills to cover the priority jobs. Version them. Test each one against real inputs before shipping.

n8n workflows (Layer 3)

Wire the skills to triggers. Schedules, webhooks, APIs, approvals. Everything visible in n8n so you can edit it yourself later.

Agent surfaces (Layer 4)

Slack bot, SMS sender, voice endpoint, or custom dashboard. We ship the one that fits where your team already works.
